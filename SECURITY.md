# Security Policy

## Supported version

Security fixes are applied to the current `main` branch and the production deployment built from it.

## Reporting a vulnerability

Please report vulnerabilities privately through GitHub's **Report a vulnerability** security-advisory form for this repository. Do not open a public issue or include credentials, payment data, customer details, or exploit code in a public discussion.

Include the affected URL or component, reproduction steps, impact, and any suggested mitigation. Credentials or tokens exposed during testing must be revoked immediately rather than sent with the report.
